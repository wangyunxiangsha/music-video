# Claudio FM — 个人 AI 音乐电台 需求文档

## 项目描述
复刻 Claudio 个人 AI 电台系统：读懂听歌习惯 → 规划声音 → 像 DJ 那样播报

## 核心功能
1. 播放器界面（PWA，可安装到手机和桌面）
2. AI DJ 播报员（DeepSeek 驱动，轻松复古风格）
3. 音乐播放（网易云音乐 NeteaseCloudMusicApi）
4. 浏览器内置 TTS 语音播报（Web Speech API）
5. WebSocket 实时状态同步
6. SQLite 本地状态持久化
7. 与 DJ 聊天功能

## 技术架构（四层）

### 第一层 — 外部上下文
- **用户品味语料**：user/taste.md、user/routines.md
- **BRAIN**：DeepSeek API（OpenAI 兼容接口）
- **MUSIC**：NeteaseCloudMusicApi（子进程，端口 3001）
- **VOICE I/O**：浏览器 Web Speech API（TTS）

### 第二层 — 本地大脑（Node.js）
- `server/index.js` — Express 主服务 + WebSocket
- `server/music.js` — 网易云 API 封装
- `server/ai.js` — DeepSeek 适配器
- `server/context.js` — 提示词组装
- `server/stats.js` — SQLite 状态记忆

### 第三层 — 运行时拼合
- Context Window = dj-persona + 用户品味 + 时间环境 + 历史记录
- Model → {say, play[], reason} → TTS 合成 → WS 推送

### 第四层 — 交互表层
- Progressive Web App（端口 8080）
- 复古电台 UI 设计
- HTTP 接口：POST /api/chat、GET /api/now、GET /api/next、WS /stream

## 用户需求确认（2026-05-17）
- 大模型：DeepSeek（需配置 DEEPSEEK_API_KEY）
- 音乐源：网易云音乐免费 API
- TTS：浏览器内置（免费，无需 Key）
- 部署：PWA（可安装到手机）
- DJ 风格：轻松复古风格

## 设计风格
- 复古电台美学：暗色系（深棕/黑）+ 金色/琥珀色
- 黑胶唱片旋转动画
- DJ 对话框（打字机效果）
- 复古字体和控件

## API 接口
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/now | 当前播放状态 |
| POST | /api/next | 切换下一首 |
| GET | /api/music/url/:id | 获取音乐播放地址（代理） |
| GET | /api/music/lyric/:id | 获取歌词 |
| GET | /api/music/search | 搜索音乐 |

## 改进计划（待办）

### 下一阶段产品路线
目标：把 Claudio FM 从“会说话的播放器”推进为“懂场景、会记忆、可轻量调教的私人音乐电台”。

核心用户需求：
- 少操作：打开就能听，点歌和切歌不需要复杂配置。
- 符合当下状态：根据时间、天气、情绪、工作/休息场景自动调整音乐。
- 有陪伴感：DJ 不只是报歌名，而是用合适的频率和语气陪伴。
- 不被打断：版权失败、切歌延迟、TTS 错位、音量突变都要尽量被系统自动消化。
- 可控但不复杂：支持“多放这个”“别放这类”“安静一点”“来点深夜模式”这类自然语言控制。

推荐实施顺序：
1. **喜欢 / 不喜欢反馈系统**：建立偏好闭环，记录“喜欢、少放、屏蔽、完整听完、频繁跳过”等信号。
2. **场景电台模式**：在现有类型播放上升级为深夜、工作、通勤、下雨、睡前、回忆杀、KTV 等场景。
3. **DJ 播报策略控制**：按场景控制 DJ 说话频率、长度和语气，例如工作时少说、深夜可多一点陪伴。
4. **队列预览和插队**：展示接下来几首歌，支持删除下一首、插队点歌、重新生成队列。
5. **每日 / 时段自动电台**：结合 `user/routines.md`、天气和历史偏好生成每日开场与时段推荐。

### 数据同步增量改造设计（已实施）

目标：把网易云 / QQ 音乐歌单同步从“全量覆盖”升级为“增量合并”，保证平台歌单能同步到最新，同时不冲掉 Claudio 本地做过的收藏、删除和屏蔽决策。

数据边界：
- `netease.playlists` / `qq.playlists`：作为平台歌单镜像，反映网易云 / QQ 官方歌单当前状态。
- `claudio.playlists`：作为 Claudio 本地收藏池，保存 `SAVE` 加入的外部推荐或恢复歌曲。
- `claudio.removedTracks`：作为最高优先级长期屏蔽表。只要歌曲在这里，自动队列就必须过滤；平台歌单同步不能自动解除屏蔽。

同步规则：
- 新同步到的平台歌单按平台和歌单 ID 合并。
- 新歌曲加入对应平台歌单；已存在歌曲更新歌名、歌手、专辑、QQ `mediaMid` 等元数据。
- 平台歌单中已经不存在的歌曲，从对应平台镜像中移除。
- `claudio.playlists` 和 `claudio.removedTracks` 原样保留，不被平台同步覆盖。
- 如果一首已屏蔽歌曲又被用户加回网易云 / QQ 官方歌单，仍然保持屏蔽；只有在 Claudio 播放器里对这首歌点 `SAVE`，才从 `removedTracks` 移除并恢复播放资格。

实现方向：
- 将合并保存逻辑集中到 `server/import.js`，提供统一 helper，例如 `mergeImportedPlaylists(existing, imported)`。
- `/api/import-playlists` 与 `npm run sync` 共用同一套合并规则，避免界面导入和命令行同步行为不一致。
- 同步结果返回更清晰的报告：新增歌曲数、更新歌曲数、平台移除歌曲数、保留屏蔽数，以及网易云 / QQ 各自成功或失败状态。
- 为增量同步补测试：保留 `removedTracks`、保留 `claudio.playlists`、平台重新加回已屏蔽歌曲仍不进入 `buildPlaylistPool()`、`SAVE` 仍可恢复屏蔽歌曲。

## Mineradio 借鉴优化计划（2026-06-28）

目标：只借鉴 Mineradio 对个人本地音乐电台有价值的部分，不复制它的桌面应用形态。重点优化 QQ 音乐 / 网易云账号登录状态、播放授权诊断、播放失败解释和本地隐私边界。

- [x] **QQ 完整播放票据识别**：保留更完整的 QQ Cookie / token 字段，识别 `qqmusic_key`、`qm_keyst`、`music_key`、`p_skey`、`skey`、`psrf_*`、`wx*` 等播放相关票据。
