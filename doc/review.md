# Claudio FM — 代码审查记录

## 2026-05-17 阶段审查：歌单导入、UI 改版、端口回退

### 审查范围

- 网易云音乐与 QQ 音乐歌单同步流程
- Claudio 暗色播放器 UI
- 播放态/停止态频谱渲染
- DJ 头像资源引用
- TTS 参数配置
- 启动端口占用处理

### 主要变更

| 模块 | 变更 |
|------|------|
| 歌单数据 | 将网易云和 QQ 音乐歌单统一同步到 `data/playlists.json` |
| QQ 音乐 | 适配用户创建歌单列表与歌单详情接口 |

| TTS | 声音、语速、音高、音量支持环境变量 |
| 启动 | 端口占用时自动尝试后续端口 |

### 审查结论

通过。当前修改符合项目目标，关键路径已有基本验证：

- `node --check public/app.js` 通过。
- `node --check server/index.js` 通过。
- 本地服务资源 `/`、`/style.css`、`/app.js`、`/avatar.jpg` 可访问。
- 模拟 8080 被占用时，服务可自动启动到 8081。

### 风险与注意事项

- UI 视觉改动主要依赖人工刷新验证，尚未建立自动截图回归。
- QQ 音乐接口依赖 Cookie 与网页请求参数，后续平台变化可能导致同步失败。
- `public/avatar.jpg` 来自 `source/头像.jpg` 的复制，后续若替换头像需同步更新 public 资源。
- MIC 目前仅提示未接入语音输入，不应在产品说明中描述为已完成语音输入功能。
- 端口回退解决启动崩溃，但用户仍需看控制台实际地址，后续可考虑启动脚本自动打开浏览器。

### 后续建议

- 增加 Playwright 或浏览器截图检查：暂停态、播放态、窄屏移动端各一张。
- 将歌单同步结果加上摘要日志，例如平台、歌单数、歌曲数、失败原因。
- 将端口回退后的地址写入 `data/server-url.txt` 或在启动日志中更醒目显示。
- 为 QQ 音乐 Cookie 失效增加明确错误提示。
