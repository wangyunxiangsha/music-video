当前阶段的重点已经完成：让电台具备稳定播放、个人歌单记忆、偏好反馈、场景队列、每日简报和 75% 本地歌单 / 25% 外部推荐的探索能力。

## 当前已知风险

- QQ 音乐仍受 Cookie 权限、平台风控、单曲版权、CDN 和签名 URL 影响。即使账号有会员，也可能出现单首歌临时不可播。
- 网易云部分歌曲只能获取试听片段，已能自动跳过，但仍会影响点歌成功率。
- 外部推荐来自搜索候选，可能存在同名歌、翻唱版、现场版或不符合预期的版本。
- 每日简报和 DJ 播报依赖 DeepSeek；API 不可用时会降级到本地模板或空播报。
- MIC 输入依赖浏览器支持和用户授权。
- 当前偏好学习主要基于本地 JSON，适合个人使用，但还不是可迁移、多用户或云同步架构。
- 播放队列的 75/25 混合策略已经可用，但还缺少用户可视化调节入口。

4. 播放更稳：在已完成连续失败保护的基础上，继续减少重复失败歌曲进入队列。

## 2026-06-29 账号授权诊断与推荐稳定性阶段

本阶段重点从“继续堆功能”转为“让个人电台更稳、更透明、更少打扰”。参考 Mineradio 后，Claudio FM 没有照搬桌面应用形态，而是吸收它在 QQ / 网易云账号登录与状态诊断上的思路：账号授权留在本机，启动不强制登录，只有点歌、会员歌或播放授权失效时才提示刷新。

### 已完成成果

- QQ 播放票据识别增强：保留 `qqmusic_key`、`qm_keyst`、`music_key`、`p_skey`、`skey`、`psrf_*`、`wx*` 等播放相关字段。
- QQ vkey 请求带播放授权：具备 music key 时使用 `ct: 19` 并携带 `authst`，提高普通会员账号拿到完整播放地址的概率。
- QQ 普通会员音质策略：高权限档位连续空 `purl` 后，临时优先尝试 `M128` / `C128` 等更稳定档位。
- 双平台账号状态面板：`SET` 集中展示 QQ 登录、QQ 播放授权、QQ Cookie 健康、网易云登录和网易云可播状态。
- SET 账号排查中心：新增 `CHECK` 重新检测按钮和诊断建议，帮助判断下一步是扫码刷新、检查权益、换源还是查看调试接口。
- 网易云扫码登录：支持在 `SET` 面板扫码刷新 `NETEASE_COOKIE`，运行中的音乐 API 子进程会使用新 Cookie。
- 队列预检查：生成或重建队列后默认预检前 5 首，提前跳过短期屏蔽、不可播、`empty purl` 或 CDN 拒绝候选。
- 推荐信号权重：新增推荐打分模块，将近期常听、喜欢、跳过、场景反馈和歌手重复控制合成权重，并写回推荐原因和分数。
- 文档边界说明：README 与部署文档已补充本地账号权益、隐私边界、Cookie 排查和扫码刷新说明。

### 当前定位

Claudio FM 现在更接近“本地个人电台控制台”：它可以导入个人歌单、自动生成混合队列、解释播放原因、按场景和反馈调音，并在平台授权不稳定时给出可排查路径。项目仍然坚持个人使用边界，不重新分发音乐，不缓存下载音频，不把 Cookie / API Key 写入备份。

### 下一步建议

1. **推荐解释 UI 化**（已完成首版）：队列条目会用 `WHY` 小标签展示推荐原因，悬停可查看包含推荐分的完整提示；“为什么播这首”回答会补充推荐分。
2. **播放完成度信号**（已完成首版）：`savePlay` 继续表示开始播放，新增 `playbackEvents` 记录半程播放、自然播完和快速手动跳过，推荐权重会轻微学习这些信号。
3. **账号诊断一键报告**：在 `SET` 面板汇总 `/api/account-status`、`/api/health`、`/api/debug/qq-circuit` 的关键字段，方便复制排查。
4. **队列预检查体验优化**：队列生成较慢时给前端一个轻提示，避免用户误以为卡住。
5. **阶段性审查与提交**：在提交前补一次 `doc/review.md`，重点看账号 Cookie 边界、队列预检查等待时间和推荐权重是否过度影响本地歌单。
