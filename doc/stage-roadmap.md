## 2026-05-20 阶段小结

本阶段 Claudio FM 已完成从“能播放和同步歌单”到“可连续运行、可按场景推荐、可自然语言调度”的核心闭环。当前系统已经可以导入网易云 / QQ 音乐歌单，维护本地播放历史和偏好反馈，按 75% 我的歌单 / 25% 外部推荐生成队列，并在播放失败时自动诊断、跳过和重建队列。

本阶段新增和修复的重点包括：

- 完成每日 / 时段自动电台简报，会结合日期、时间、天气、近期播放和偏好生成开场。
- 完成队列折叠 / 展开能力，并修复 Chrome 与 Firefox 因本地状态不同导致展示不一致的问题。
- 完成 75% 本地歌单 / 25% 外部推荐混合队列，外部推荐会参考时间、场景和常听风格。
- 完成 QQ 音乐播放失败保护：URL probe、不可播缓存、circuit breaker、同源音频代理、stream 中断保护和更准确的前端失败文案。
- 完成连续失败保护与播放失败诊断接口，连续失败时会重建队列，并可通过调试接口查看失败原因。
- 完成“上午我想听轻音乐”“阴天来点轻音乐”“下午想听国风”这类时间 + 风格指令，AI 电台会自动安排对应风格队列。
- 完成播放历史 `HIST` 面板和队列 `QUEUE` 区域的可见性修复，避免浏览器缓存或本地折叠状态造成误判。

当前更适合进入“推荐质量、用户控制、长期偏好记忆”阶段，而不是继续堆叠播放器基础能力。

## 当前阶段定位

Claudio FM 现在已经从基础播放器推进到“个人 AI 音乐电台”阶段：它不只是能播放歌曲，还能导入用户歌单、理解简单自然语言指令、根据偏好和场景生成队列，并用 DJ 播报形成陪伴感。

当前阶段的重点已经完成：让电台具备稳定播放、个人歌单记忆、偏好反馈、场景队列、每日简报和 75% 本地歌单 / 25% 外部推荐的探索能力。

## 已完成成果

### 1. 播放器与电台界面

- 完成 Claudio 暗色复古电台 UI，包含顶部时间、播放面板、频谱、进度条、音量、DJ 区域和底部快捷输入。
- 支持 PWA 安装，包含 manifest 和 service worker。
- 修复 service worker 缓存旧页面的问题，HTML / JS / CSS 更新后能更及时生效。
- `QUEUE` 区域支持折叠和展开，折叠状态保存到浏览器 `localStorage`。

### 2. 音乐源与歌单导入

- 接入网易云音乐 API，支持搜索、播放 URL、歌词、用户歌单同步。
- 接入 QQ 音乐，支持搜索、歌词、用户创建歌单同步和播放 URL 获取。
- 本地沉淀 `data/playlists.json`，为后续分类、推荐和队列生成提供基础数据。
- 支持将歌单歌曲分类，生成 `data/categories.json` 和 `data/songs-classified.json`。

### 3. 播放稳定性

- 对网易云试听片段做检测，短试听自动切下一首。
- QQ 音乐 URL 获取加入 CDN probe，避免拿到看似存在但实际 404 的链接。
- QQ URL 连续失败时触发 circuit breaker，减少长时间阻塞。
- 自动队列切歌前加入可播校验，跳过暂不可播候选。
- QQ 不可播 `songmid` 加入短期缓存，避免反复探测同一首失败歌曲。
- QQ 音频改为服务端同源代理，减少浏览器直连 QQ CDN 的跨域、请求头和缓存差异。
- 音频代理已处理上游 stream 中途断开，单首歌失败应退化为提示或自动切歌，不应导致服务进程崩溃。
- 前端播放失败文案已从“没有可播放版权”改为“当前音源暂时打不开”，避免误判用户账号权限。

### 4. AI DJ 与语音体验

- 接入 DeepSeek，用于 DJ 播报、聊天、每日简报和用户品味摘要。
- 服务端 Edge TTS 已接入，支持通过 `.env` 调整音色、语速、音调和音量。
- 前端播放新歌、切歌、失败切歌时会停止旧 DJ 打字机和 TTS，避免播报错位。
- 支持浏览器 MIC 语音输入，识别后复用 DJ 文本指令流程。

### 5. 偏好记忆与推荐

- 播放历史会写入本地 `data/stats.json`。
- 底部 `HIST` 面板展示播放记录数、去重歌曲、常听歌手、常听类型和最近播放。
- 支持 `LIKE` / `LESS` 控件和自然语言反馈，例如“喜欢这首”“少放这个歌手”“别放这类”。
- 反馈会影响本地推荐排序，并过滤被明确屏蔽的歌曲、歌手或类型。
- 智能扩展队列已按 75% 本地歌单 / 25% 外部推荐混合，外部推荐会参考时间、场景、近期常听歌手和常听类型。

## 当前已知风险

- QQ 音乐仍受 Cookie 权限、平台风控、单曲版权、CDN 和签名 URL 影响。即使账号有会员，也可能出现单首歌临时不可播。
- 网易云部分歌曲只能获取试听片段，已能自动跳过，但仍会影响点歌成功率。
- 外部推荐来自搜索候选，可能存在同名歌、翻唱版、现场版或不符合预期的版本。
- 每日简报和 DJ 播报依赖 DeepSeek；API 不可用时会降级到本地模板或空播报。
- MIC 输入依赖浏览器支持和用户授权。
- 当前偏好学习主要基于本地 JSON，适合个人使用，但还不是可迁移、多用户或云同步架构。
- 播放队列的 75/25 混合策略已经可用，但还缺少用户可视化调节入口。

## 下一阶段目标

下一阶段建议目标：把 Claudio FM 从“能自动推荐的个人电台”推进到“更稳定、更可控、更懂长期偏好的私人音乐伙伴”。

重点不再是堆新功能，而是提高三个质量指标：

1. 推荐更准：外部推荐不突兀，能逐步学习用户真实偏好。
2. 控制更自然：用户能用少量语言或轻量控件调整电台方向。
3. 状态更透明：用户能知道为什么放这首、当前是什么场景、外部推荐占比是多少。
4. 播放更稳：在已完成连续失败保护的基础上，继续减少重复失败歌曲进入队列。

## 阶段完成判定

- 连续播放 30 分钟内，单曲失败不会导致服务退出。
